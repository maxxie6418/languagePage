请帮我设计并开发一个「考研英语学习平台」网页应用，核心目标是：围绕中国考研英语大纲，提供系统化、高信息密度、长期学习型的英语学习工具。整体风格专业、克制、现代，偏学习工具而非营销网站。界面应强调效率、可读性与长期使用体验。

# 一、项目定位

这是一个专注于「考研英语」的学习网站，目标用户为准备考研英语（一/二）的大学生。

网站核心原则：

- 信息密度高
- 学习路径清晰
- 操作效率高
- 长时间阅读舒适
- 学习进度可视化
- 更像 Notion + Anki + 真题平台 的结合体
- 不做花哨动画，不做营销风
- 偏桌面端优先设计

整体气质参考：

- Obsidian
- Notion
- Anki
- Duolingo 的学习反馈机制
- GitHub 文档式布局
- 高级教育平台 UI

------

# 二、网站结构

网站分为三大主模块：

1. 基础学习
2. 真题解析
3. 模拟测验

顶部为固定导航栏。

左侧为学习导航树。

中间为主内容区。

右侧为学习辅助区（笔记、收藏、掌握度、统计等）。

整体布局类似：

- 左：目录导航
- 中：学习内容
- 右：学习状态与工具栏

# 三、模块设计

# 1. 基础学习模块（核心）

这是信息量最大的模块。

需要包含以下子模块：

## 1.1 单词学习

核心功能：

### 单词列表

按照：

- 考研英语大纲词汇
- 高频词
- 低频词
- 真题核心词
- 熟词僻义
- 阅读高频词

分类。

每个单词卡片需包含：

- 英文单词
- 音标
- 中文释义
- 词性
- 常见搭配
- 例句
- 真题出现年份
- 派生词
- 同义词
- 反义词
- 词根词缀拆解
- 记忆提示

例如：

Word: address
音标：/əˈdres/
词性：v./n.
释义：

- 处理
- 演讲
- 地址

搭配：

- address the issue
- address a problem

真题年份：
2012、2015、2021

------

### 单词掌握状态系统（重点）

每个单词必须有学习状态：

- 未掌握（默认）
- 记忆中
- 已掌握

使用不同颜色和视觉标签。

支持：

- 单击切换状态
- 批量修改
- 统计数量
- 学习进度条
- 筛选查看

例如：

- 仅看未掌握
- 仅看记忆中
- 仅看已掌握

------

### 单词显示控制（重点）

必须支持：

- 隐藏中文释义
- 隐藏英文单词
- 隐藏例句
- 随机顺序
- 卡片翻转模式

适合自测。

支持：

- “我认识”
- “不认识”

快捷操作。

------

### 单词学习模式

包含：

#### 列表模式

高信息密度表格布局。

#### 卡片模式

类似 Anki。

#### 拼写模式

根据中文拼写英文。

#### 默写模式

只显示释义。

#### 真题上下文模式

单词放入考研阅读原句中学习。
